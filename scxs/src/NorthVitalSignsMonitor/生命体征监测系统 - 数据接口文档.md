# 生命体征监测系统 - 数据接口文档

## 1. 数据传输格式

### 1.1 请求数据格式
```json
{
  "deviceId": "TEST001",        // 设备ID，字符串，必填
  "timestamp": "2024-01-18 15:30:00",  // 时间戳，字符串，必填，格式：YYYY-MM-DD HH:mm:ss
  "vitalSigns": {              // 生命体征数据对象，必填
    "temperature": 36.5,       // 体温，数字，可选，单位：℃，非负数
    "bloodPressure": {         // 血压对象，可选
      "systolic": 120,         // 收缩压，数字，可选，单位：mmHg，非负数
      "diastolic": 80          // 舒张压，数字，可选，单位：mmHg，非负数
    },
    "heartRate": 75,           // 心率，数字，可选，单位：次/分，非负数
    "bloodOxygen": 98          // 血氧，数字，可选，单位：%，非负数
  }
}
```

### 1.2 数据字段说明

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|-----|------|
| deviceId | string | 是 | 设备唯一标识符 |
| timestamp | string | 是 | 数据采集时间 |
| vitalSigns | object | 是 | 生命体征数据对象 |
| vitalSigns.temperature | number | 否 | 体温值，单位：℃ |
| vitalSigns.bloodPressure | object | 否 | 血压数据对象 |
| vitalSigns.bloodPressure.systolic | number | 否 | 收缩压，单位：mmHg |
| vitalSigns.bloodPressure.diastolic | number | 否 | 舒张压，单位：mmHg |
| vitalSigns.heartRate | number | 否 | 心率，单位：次/分 |
| vitalSigns.bloodOxygen | number | 否 | 血氧饱和度，单位：% |

### 1.3 数据规则
- 所有可选字段如果提供值，必须为非负数
- 未提供的可选字段将默认显示为 0
- 时间戳必须符合指定格式
- JSON 格式必须完全符合规范，包括正确的括号匹配和逗号使用

## 2. 错误码说明

### 2.1 错误响应格式
```json
{
  "code": 1001,               // 错误码
  "message": "错误描述信息",   // 错误描述
  "timestamp": "2024-01-18 15:30:00"  // 错误发生时间
}
```

### 2.2 错误码定义

| 错误码 | 说明 | 处理建议 |
|-------|------|---------|
| 0 | 成功 | 数据格式正确，成功接收 |
| 1001 | 体温数据异常 | 检查体温值是否为负数 |
| 1002 | 血压数据异常 | 检查收缩压或舒张压是否为负数 |
| 1003 | 心率数据异常 | 检查心率值是否为负数 |
| 1004 | 血氧数据异常 | 检查血氧值是否为负数 |
| 1005 | 数据格式错误 | 检查 JSON 格式是否正确，确保所有必填字段存在 |
| 1006 | 时间戳错误 | 检查时间戳格式是否正确 |

## 3. 示例

### 3.1 正确的数据示例
```json
{
  "deviceId": "TEST001",
  "timestamp": "2024-01-18 15:30:00",
  "vitalSigns": {
    "temperature": 36.5,
    "bloodPressure": {
      "systolic": 120,
      "diastolic": 80
    },
    "heartRate": 75,
    "bloodOxygen": 98
  }
}
```

### 3.2 最小化数据示例（仅必填字段）
```json
{
  "deviceId": "TEST001",
  "timestamp": "2024-01-18 15:30:00",
  "vitalSigns": {}
}
```

## 4. 注意事项

1. 数据传输使用 TCP 协议
2. 数据编码使用 UTF-8
3. 每次发送的数据必须是完整的 JSON 对象
4. 建议在发送数据前进行本地格式验证
5. 收到错误响应时，请根据错误码进行相应处理

## 5. 调试建议

1. 使用提供的测试工具验证数据格式
2. 确保所有数值类型字段为数字，不要使用字符串形式的数字
3. 注意检查 JSON 格式的完整性和正确性
4. 对于可选字段，可以选择不发送或发送 null，系统会处理为 0
